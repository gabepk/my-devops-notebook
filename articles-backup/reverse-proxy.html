What is proxy? 

n law, a proxy (in protuguese, Procuração), is an authorization document that allows that someone can take legal/official decisions as agent of another person. The main goal of a proxy server, in computer science, is to also to represent someone/something, which can be either the client or the server.

A proxy is an intermediate level server (either hardware or software) that receives requests from a client, modifies it (add/remove headers, encrypt/decrypt message, compress/decompress message, etc) and delivers to the server.

collapsed forward

  • Proxies have the capability to coordinate several requests, by join similar requests and deliver only one to the server. This optimizes the bandwidth traffic.
  • This can be used to forward requests from servers within the same distributed system to a commom database. It can join requests for the same data or data that are physically close.
  • One of the pitfalls is that in order to collapse several requests, the first request will have to wait x seconds for it's response, where x is the time to join the requests.

What is a reverse proxy? 

A reverse proxy (aka gateway server) receives a request from a client, then it forward the request to one of several backend servers.

In Apache, a server can be configured as a reverse proxy instead of a basic web server. The ProxyPass directive specifies the mapping of incoming requests:

ProxyPass "/" "http://www.example.com/"

The ProxyPassReverse directive rewrites any URL that the backend server returns to the proxy, if it needs to. Therefore, any requests which start with "/images/" (whether it comes from the client or the backend server), will be proxied to "example.com". Otherwise it will be handled locally (on the proxy server).

ProxyPassReverse "/images/" "https://www.example.com/"
