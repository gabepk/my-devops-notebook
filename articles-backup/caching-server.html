Caching Server is a server that contains frequently-accessed data (documents, imagens, articles) available to the clients. It decreases the number of requests to the backend servers and improves performance of the distributes system.

A good cache server has more cache hit than cache miss.

 

As an example, Apache has a web proxy caching called Traffic Server. It serves cached data based on the headers of the requests, freshness of data, flags configured on the server, an so on. Examples:

  • It determines if an HTTP object is fresh checking the Expires or max-age headers.
  • as
  • To control the freshness of a data, the following variables must be edited on the file records.config:  
    proxy.config.http.cache.heuristic_min_lifetime
proxy.config.http.cache.heuristic_max_lifetime
  • It supports the HTTP PUSH method to deliver content directly into the cache without client requests.
